# Allgemeine Hinweise

## Hinweise zum Deployment
Wenn Sie Ihre App nicht nur lokal testen wollen, sondern für andere (automatisch) öffentlich verfügbar machen wollen, gibt es die folgenden kostenlosen Plattformen, die sich mit Github Actions sehr gut und einfach verknüpfen lassen.
- Spring Boot App [Render](https://render.com) z.B. mit folgendem Tutorial: [HowTo](https://hostingtutorials.dev/blog/free-spring-boot-host-with-render)
- Python App [Deta Space](https://deta.space/) z.B. mit folgendem Tutorial: [HowTo](https://deta.space/docs/en/build/quick-starts/python)
- React/Vite/Vue.js, diverse Möglichkeiten: z.B. direkt mit [Github Pages oder Vercel]( https://blog.logrocket.com/8-ways-deploy-react-app-free/)

## Entwicklungsumgebungen
Grundsätzlich steht Ihnen die Wahl Ihrer Entwicklungsumgebung (IDE) frei. Sie profitieren jedoch davon, dass alle in Ihrem Team sich auf die Nutzung einer IDE verständigen. Empfehlenswert sind [VS Code](https://code.visualstudio.com/) und [IntelliJ](https://www.jetbrains.com/idea/) (Ultimate-Version für HNU-Angehörige kostenlos) bzw. [PyCharm](https://www.jetbrains.com/pycharm/)(Professional-Version für HNU-Angehörige kostenlos).

## Dokumentation mit arc42
In dieser Datei können Sie, basierend auf dem arc42-Template, Ihr Softareprojekt strukturiert dokumentieren. Bitte beachten Sie für die Verwendung des Templates die Hinweise aus der Lehrveranstaltung. Weitere Hinweise finden Sie auf [der Website von arc42](https://docs.arc42.org/).
